# Bid Master System

ระบบช่วยบริหารงานรับเหมาก่อสร้าง — คำนวณต้นทุน และติดตามการประมูลภาครัฐ
บริหารโดย **Sebastian** (หัวหน้า Agent) ร่วมกับ Sub Agent เฉพาะทาง

---

## โครงสร้างโปรเจกต์

```
Bid-Master-System/
│
├── Agent/                  ฐานข้อมูลความรู้และบุคลิกของ Sebastian
│   ├── about work/         สถานะและความคืบหน้างานปัจจุบัน
│   └── conversations/      บันทึกการสนทนาและการตัดสินใจสำคัญ
│
├── data/                   ข้อมูลทั้งหมดของระบบ
│   ├── company/            ข้อมูลบริษัท (วางเองได้)
│   │   ├── rates/          ราคาวัตถุดิบและค่าแรง
│   │   ├── templates/      เทมเพลต Excel, BOQ, ใบเสนอราคา
│   │   └── reference/      TOR เก่า, เอกสารอ้างอิง
│   ├── bids/               TOR และข้อมูลประมูลใหม่
│   ├── raw/                ข้อมูลดิบจาก scraper (อัตโนมัติ)
│   ├── processed/          ข้อมูลหลัง Sub Agent วิเคราะห์แล้ว
│   └── archive/            ข้อมูลโครงการที่ปิดแล้ว
│
├── scripts/                สคริปต์ Python อัตโนมัติ
├── tor-reader/             เครื่องมืออ่านและวิเคราะห์ TOR
├── docs/                   เอกสารประกอบระบบ
└── backups/                สำรองข้อมูลสำคัญ
```

---

## วิธีใช้งาน

1. **วางไฟล์บริษัท** ลงใน `data/company/` ตามประเภท
2. **วาง TOR ใหม่** ลงใน `data/bids/`
3. **สั่ง Sebastian** ด้วยภาษาธรรมชาติ — Sebastian จะแบ่งงานให้ Sub Agent เอง

---

## Sub Agent

| Sub Agent | บทบาท | สถานะ |
|-----------|--------|-------|
| Sub Agent A | อ่าน TOR → วิเคราะห์วัตถุดิบ | วางแผนแล้ว |
| Sub Agent B | คำนวณต้นทุน + ราคาเสนอ | วางแผนแล้ว |
| Sub Agent C | จัดอันดับความน่าประมูล | วางแผนแล้ว |
| Sub Agent D | Supervisor ควบคุมกระบวนการ | วางแผนแล้ว |
| Sub Agent E | อัปเดตราคาวัตถุดิบอัตโนมัติ | ไอเดียใหม่ |
